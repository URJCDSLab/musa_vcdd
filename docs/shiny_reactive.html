Objetos reactivos

Reactividad

  • Lo básico: los “outputs” reaccionan ante un cambio en un “input”

  • No es suficiente a medida que aumenta la complejidad

  • Por ejemplo, si hay outputs que comparten datos, se puede “delegar” la generación de estos a un objeto reactivo

  • Objetos reactivos: reactive(), reactiveValues(), eventReactive(), reactiveVal()

  • “Lanzador” de eventos: observeEvent()

reactive()

  • Se asigna a un nombre como cualquier otro objeto

  • Después se usa llamando al objeto con paréntesis:

datos_reactivos <- reactive({
  # código para generar los datos
})
output$tabla <- renderTable({
  datos_reactivos()
})

Ejemplo

library(shiny);library(bslib);library(reactable);library(dplyr)
resultados <- read.csv("data/results.csv")
ui <- page_navbar(
  title = "Análisis resultados FIFA",
  sidebar = sidebar(
    title = "Selección",
    selectizeInput(
      "sel_pais",
      "Selecciona un país:",
      choices = unique(resultados$home_team)
    )
  ),
  nav_panel(
    title = textOutput("partidos"),
    reactableOutput("tabla_partidos"),
      verbatimTextOutput("num_partidos")
  ),
  nav_panel(title = "Goleadores")
)

server <- function(input, output, session) {
    num <- reactive({
    resultados |>
      filter(home_team == input$sel_pais) |>
      nrow()
  })
  output$tabla_partidos <- renderReactable({
    resultados |>
      filter(home_team == input$sel_pais) |>
      reactable()
  })

  output$num_partidos <- renderPrint({
    paste("Número de partidos:", num())
  })
  output$partidos <- renderText({
    paste0("Partidos (", num(), ")")
  })
}

reactiveValues()

  • Permite crear un objeto reactivo que puede contener múltiples valores
  • Se asignan valores con $ y se accede a ellos de la misma forma
valores <- reactiveValues(a = 1, b = 2)
valores$a <- 3
output$valor_a <- renderText({
  paste("Valor a:", valores$a)
})

reactiveVal()

  • Permite crear un objeto reactivo que contiene un solo valor

  • Se asigna el valor con paréntesis y se accede de la misma forma

contador <- reactiveVal(0)
contador(contador() + 1)
output$contador <- renderText({
  paste("Contador:", contador())
})

eventReactive() y observeEvent()

  • Permiten ejecutar código reactivo en respuesta a eventos específicos, como un clic en un botón

  • eventReactive() devuelve un valor reactivo que se actualiza cuando ocurre el evento

resultado <- eventReactive(input$boton, {
  # código para generar el resultado
})
output$resultado <- renderText({
  resultado()
})
  • observeEvent() ejecuta código sin devolver un valor, útil para acciones como mostrar mensajes
contador <- reactiveVal(0)
observeEvent(input$boton, {
  contador(contador() + 1)
})
output$contador <- renderText({
  paste("Contador:", contador())
})

req()

  • Función que se utiliza para requerir que ciertas condiciones se cumplan antes de ejecutar un bloque de código reactivo
  • Si la condición no se cumple, el bloque de código no se ejecuta y se muestra un mensaje de error o se mantiene el estado anterior
output$resultado <- renderText({
  req(input$texto != "") # Requiere que el input de texto no esté vacío
  paste("El texto ingresado es:", input$texto)
})

validate()

  • Función que se utiliza para validar los inputs antes de ejecutar un bloque de código reactivo
  • Permite mostrar mensajes de error personalizados si los inputs no cumplen con ciertos criterios
output$resultado <- renderText({
  validate(
    need(input$numero > 0, "El número debe ser mayor que cero")
  )
  paste("El número ingresado es:", input$numero)
})

Carga y descarga de ficheros

  • fileInput(): input para subir archivos, con opciones de tipo de archivo, múltiples archivos, etc.

  • En el server, se asigna a objetos que se pueden usar

<|input|>fileInput("archivo", "Subir archivo CSV", accept = ".csv")

output$archivo <- renderTable({
  req(input$archivo) # Requiere que se haya subido un archivo
  read.csv(input$archivo$datapath) # Lee el archivo subido
})
  • downloadHandler(): permite crear un enlace para descargar archivos generados en la aplicación
<|input|>downloadButton("descargar", "Descargar datos")

output$descargar <- downloadHandler(
  filename = function() {
    paste("datos-", Sys.Date(), ".csv", sep = "")
  },
  content = function(file) {
    write.csv(datos_reactivos(), file) # Escribe los datos reactivos en un archivo CSV
  }
)

library(shiny);library(bslib);library(reactable);library(dplyr);library(ggiraph);library(ggplot2)
ui <- page_navbar(
  title = "Análisis resultados FIFA",
  sidebar = sidebar(
    title = "Selección",
    fileInput("archivo", "Subir archivo CSV", accept = ".csv"),
    uiOutput("sel_pais_ui"),

server <- function(input, output, session) {

        read.csv(input$archivo$datapath)

      "sel_pais",

      choices = unique(resultados()$home_team)

      filter(home_team == input$sel_pais) |>

    validate(need(input$sel_rival, "Se requiere un rival. Carga los rivales y selecciona uno para mostrar el gráfico."))
    p <- resultados() |>
      filter(home_team == input$sel_pais, away_team == input$sel_rival) |>
      ggplot(aes(x = date, y = home_score)) +
      geom_line() +
      geom_point_interactive(aes(tooltip = paste("Fecha:", date, "<br>Marcador:", home_score, "-", away_score)))
    girafe(ggobj = p)
  })

  output$descargar <- downloadHandler(
  filename = function() {
    paste("datos-", Sys.Date(), ".csv", sep = "")
  },
  content = function(file) {
    write.csv(resultados(), file) # Escribe los datos reactivos en un archivo CSV
  }
)

}

Gráficos interactivos

Gráficos interactivos con ggiraph

  • Permite crear gráficos interactivos a partir de gráficos ggplot2

  • Alternativa a plotly (ha dejado de soportar R)

  • Opción sencilla: añadir “*_interactive” a las funciones de ggplot2 (e.g., geom_point_interactive(), geom_bar_interactive(), etc.)

  • Múltiples opciones y personalización: tooltips, acciones al hacer clic, ejes, etc.

  • Libro: https://www.ardata.fr/ggiraph-book/

Ejemplo simple

library(shiny)
library(palmerpenguins); library(ggiraph); library(ggplot2); library(dplyr)
ui <- fluidPage(title = "Simple demo app",
                sidebarLayout(
                  sidebarPanel(  
                    varSelectInput("varx", "Explanatory variable", penguins |> select(where(is.numeric))),
                    varSelectInput("vary", "Response variable", penguins |> select(where(is.numeric))),
                    varSelectInput("varcol", "Colour variable", penguins |> select(where(is.factor)))),
                  mainPanel( 
                    girafeOutput("plot"),
                    verbatimTextOutput("model")))
)
server <- function(input, output, session){
  output$plot <- renderGirafe({
    p <- penguins |> ggplot(aes_string(x = input$varx, y = input$vary, col = input$varcol)) +
      geom_point_interactive(aes(tooltip = paste("X:", get(input$varx), "<br>Y:", 
      get(input$vary), "<br>Colour:", get(input$varcol))))
    girafe(ggobj = p)
  })
  output$model <- renderPrint({
    m <- lm(get(input$vary) ~ get(input$varx) + get(input$varcol), penguins)
    summary(m)})
}
shinyApp(ui = ui, server = server)

Mapas interactivos con {leaflet}

  • Permite crear mapas interactivos con múltiples capas, marcadores, polígonos, etc.

  • Necesaria columna “geometry” con objetos sf (coordenadas, polígonos, etc.)

  • Para datos puntuales, puede ser coordenadas x e y solamente

  • Para datos deportivos, puede ser útil para mostrar ubicaciones de eventos, instalaciones deportivas, etc.

Mapas interactivos con {ggiraph}

  • Alternativa interactiva a los mapas de {ggplot2} con geom_sf_interactive()

  • Sin necesidad de usar {leaflet}, pero con menos opciones de personalización y sin capas dinámicas

Ejemplo simple con {leaflet}

library(shiny); library(leaflet); library(sf); library(dplyr)
countries_data <- giscoR::gisco_get_countries() |> 
  select(NAME_ENGL, geometry)
resultados <- read.csv("data/results.csv") |> 
  group_by(home_team) |> 
  summarise(total_partidos = n()) |>
  inner_join(countries_data, by = c("home_team" = "NAME_ENGL")) |> 
  st_as_sf()
ui <- fluidPage(
  titlePanel("Mapa interactivo con Leaflet"),
  sidebarLayout(
    sidebarPanel(
      selectizeInput("pais", "Selecciona un país:", choices = unique(resultados$home_team))
    ),
    mainPanel(
      leafletOutput("mapa")
    )
  )
)
server <- function(input, output, session) {
  output$mapa <- renderLeaflet({
    bins <- c(quantile(resultados$total_partidos, probs = seq(0, 1, 0.25)), Inf)
    pal <- colorBin("YlOrRd", domain = resultados$total_partidos, bins = bins)
    # Cargar datos geoespaciales (ejemplo con sf)
    resultados |> 
    leaflet() |> 
      addTiles() |> 
      addPolygons(fillColor = ~pal(total_partidos), 
        fillOpacity = 0.5, 
        color = "black", 
        weight = 1, 
        popup = ~paste0("Total partidos: ", total_partidos), 
        label = ~home_team) |> 
      addPolygons(data = resultados |> filter(home_team == input$pais), 
        color = "green", weight = 4) |> 
      addLegend(pal = pal, values = resultados$total_partidos, title = "Total partidos")
  })
}
shinyApp(ui = ui, server = server)
